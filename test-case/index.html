Select box [disabled] snapshot test

To run test do: 

npm install
npm run test
# or npm run test -- --ui to show UI

Testcase with a <select> element where one of the values has the selected prop set.

See test file in tests/example.spec.js

Expected

When using /children: deep-equal in a combobox snapshot, the snapshot should not match if the snapshot and the actual aria tree do not have identical [selected] flags.

Actual

The snapshot matches both when the [selected] props is present or missing. It does however not match if there is a [selected] property on the wrong select element in the snapshot.
